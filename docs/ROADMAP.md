# Roadmap

## Phase 0 — Project setup

- Create Next.js app
- Add Tailwind CSS
- Add shadcn/ui
- Add layout shell
- Add sidebar navigation
- Add mock data
- Add page routes

---

## Phase 1 — Static/manual MVP

Goal: dashboard works with sample data and manual editing.

Build:

- Main dashboard
- Bookings page
- Guests page
- Tasks page
- Reviews page
- Finance page
- Marketing & AI page
- Seed data
- Local state or simple database persistence

---

## Phase 2 — Supabase persistence

Goal: make the MVP usable with real manual data.

Build:

- Supabase schema
- Auth
- CRUD for bookings
- CRUD for guests
- CRUD for tasks
- CRUD for reviews
- CRUD for campaigns
- Activity log

---

## Phase 3 — AI features

Goal: make the dashboard genuinely helpful.

Build:

- Daily briefing generator
- AI suggestion card
- Booking gap detector
- Campaign draft generator
- Review reply drafter
- Social post generator
- Email draft generator

---

## Phase 4 — Import/export

Goal: bridge to real-world data without full API complexity.

Build:

- CSV import for bookings
- CSV import for guests
- CSV import for marketing metrics
- CSV export for reports

---

## Phase 5 — Read-only integrations

Goal: pull real data into the dashboard.

Build:

- Cloudbeds read-only sync
- GA4 metrics sync
- Search Console sync
- Google Business Profile review/performance sync
- Google Ads reporting sync

---

## Phase 6 — Controlled actions

Goal: let AI create drafts or paused assets.

Build:

- Create paused Google Ads campaign
- Create email draft in email platform
- Save website/social content drafts
- Approval workflow
- Action audit log

---

## Phase 7 — Automation with guardrails

Goal: safe, human-approved recommendations.

Build:

- Weekly marketing recommendations
- Budget-change suggestions
- Campaign optimisation suggestions
- Automatic review request task creation
- Recurring maintenance tasks
