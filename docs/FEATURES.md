# Feature Specification

## 1. Main dashboard

The main dashboard is the primary screen.

### Purpose

Show the owner what matters today and what to do next.

### Cards

#### Today at a glance

- Check-ins today
- Check-outs today
- Current guests
- Occupancy this month
- Revenue this month
- Review score

#### Upcoming bookings

Show the next 5–10 bookings:

- Date
- Guest name
- Room/stay type
- Number of nights
- Status

#### Occupancy calendar

A compact 30-day calendar with colour status:

- Booked
- Available
- Gap
- Blocked

#### Today’s tasks

Checklist of tasks due today:

- Prepare room for check-in
- Stock breakfast provisions
- Clean and inspect rooms
- Check firewood levels
- Reply to guest messages

#### Guest messages

MVP can use manual/sample messages.

Future integration options:

- Email
- SMS
- WhatsApp
- Cloudbeds messages
- Airbnb/OTA messages if available

#### Review requests

Track guests who have checked out and should receive a review request.

Statuses:

- Not ready
- Ready to send
- Sent
- Review received
- No response

#### Marketing overview

Basic marketing metrics:

- Website visitors
- Enquiries
- Booking clicks
- Conversion rate

#### AI suggestion

One high-priority suggestion:

Examples:

- Create mid-week campaign for booking gap
- Send review requests
- Draft social post
- Follow up unbooked enquiries
- Improve landing page

## 8. Google Ads campaign builder later

The Google Ads builder should be human-approved.

MVP version:

- AI creates campaign draft
- User edits and saves
- No live publishing

Later version:

- Create paused campaigns through Google Ads API
- User approves before launch
- AI analyses performance and suggests improvements
